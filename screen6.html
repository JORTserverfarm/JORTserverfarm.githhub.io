Improvements & Future Work

While the Raspberry Pi server cluster is fully functional, there are several areas where it could be enhanced for better performance and usability.

Network Performance

The current switch limits the startup speed of workloads. Upgrading to a higher-quality, faster switch would allow jobs to launch significantly quicker.

Network Redundancy

A redundant switch has been included in the system design to reduce single points of failure and improve overall reliability. This would allow for failover in the event of a primary switch issue, helping to maintain network connectivity across the cluster. However, due to the lack of a USB-to-Ethernet adapter, it is not currently implemented and instead remains in place to showcase the intended functionality.

Power & Scalability

Power-over-Ethernet (PoE) was not implemented due to resource constraints, but the design supports it without modification. Implementing PoE in the future would simplify power management and make scaling the cluster easier.

These improvements focus on faster workload handling, easier management, and better scalability without requiring major redesigns.
