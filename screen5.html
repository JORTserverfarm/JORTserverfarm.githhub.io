System Overview

This project is a Raspberry Pi–based server cluster designed to explore scalable, low-power compute infrastructure. The system consists of a dedicated control node and four diskless worker nodes, all mounted inside a custom aluminum frame.

Instead of operating as individual computers, each Raspberry Pi functions as part of a single, coordinated system. Network configuration, firmware, and workloads are all managed centrally, allowing the cluster to behave like a small-scale data center.

Head Node & Network Control

The Head Pi is the most critical component of the cluster and operates exclusively as the control plane. It runs DHCP and TFTP services, dynamically assigning IP addresses and boot images to each worker node as they come online.

All worker nodes are fully PXE-booted and diskless, meaning they contain no permanent operating system or storage. This allows nodes to be added, removed, or replaced without reconfiguration.

Centralized booting makes the cluster highly modular and simplifies both maintenance and scalability.

Storage & Workload Management

A 2 TB external high-speed drive connected to the Head Pi provides shared storage for AI models and firmware. Worker nodes access this data over the network, eliminating the need for local disks while maintaining flexibility.

The cluster runs k3s, a lightweight Kubernetes distribution, which automatically schedules AI workloads across the available worker nodes. This ensures balanced compute usage and maintains high availability even if individual nodes are restarted.

Access, Cooling, and Future Improvements

A dedicated Access Pi provides immediate local terminal access to the cluster, allowing administrators to monitor and manage the system directly without relying on external devices.

Thermal management is handled by two large fans mounted to the rack, powered directly from the Raspberry Pis. This ensures continuous airflow across all nodes during sustained workloads.

While Power-over-Ethernet was not implemented due to resource constraints, the physical design supports PoE with no structural changes. Network performance is currently limited by the existing switch, and upgrading to a faster model would significantly improve AI workload startup times.

Future improvements also include a higher-quality display to make on-device monitoring and management more intuitive.
